# ISO 26262 vs ISO 25119 機能安全規格の違い

建設機械開発における機能安全規格の整理。自動車業界出身の委託先とのコミュニケーションで共通言語となる比較知識をまとめる。

---

## 1. 背景：なぜ2つの規格があるのか

機能安全の国際基本規格 **IEC 61508**（Functional Safety of E/E/PE Safety-related Systems）を産業・自動車・農業機械など各分野に特化させた派生規格が存在する。

```
IEC 61508（基本規格）
    │
    ├── ISO 26262  ── 乗用車・商用車
    │
    ├── ISO 25119  ── 農業・林業・建設機械
    │
    ├── IEC 62061  ── 産業機械
    │
    └── その他分野別派生規格
```

建設機械はISO 25119が直接適用されるが、ISO 26262の知識を持つ委託先・エンジニアと協働する場面では差異を把握しておくことが重要。

---

## 2. 規格の概要比較

| 項目 | ISO 26262 | ISO 25119 |
|---|---|---|
| 正式名称 | Road vehicles — Functional safety | Tractors and machinery for agriculture and forestry — Safety-related parts of control systems |
| 対象 | 量産乗用車・バス・トラック等（最大3.5トン以下が主） | 農業機械・林業機械・建設機械（掘削機・クレーン等） |
| 初版 | 2011年（第2版：2018年） | 2010年（第2版：2020年） |
| ハザードリスク分類 | ASIL（A〜D） | AgPL（a〜e）またはSIL（1〜3） |
| 開発プロセス要件 | V字モデルに基づく厳密なプロセス要件 | プロセス要件あり（ISO 26262ほど詳細ではない） |
| 量産前提 | 大量生産（万〜百万台規模）前提 | 少量生産・多品種前提 |
| 既存機械対応 | 新規開発中心（改造車両の扱いあり） | 既存機械（Proven-in-Use）の扱いを明示 |

---

## 3. 安全度要求水準の分類

### 3.1 ASIL（ISO 26262）

ASIL（Automotive Safety Integrity Level）：A・B・C・D の4段階（DがもっともリスクHighを意味する安全要求）。

ASILの決定には HARA（Hazard Analysis and Risk Assessment）で以下の3パラメータを評価する。

| パラメータ | 意味 | 例（自動車） |
|---|---|---|
| S（Severity） | ハザード発生時の傷害の深刻度（S0〜S3） | S3: 生命の危険 |
| E（Exposure） | ハザード状況への暴露頻度（E0〜E4） | E4: 常時暴露 |
| C（Controllability） | ドライバーによる回避可能性（C0〜C3） | C3: 回避困難 |

```
ASIL = f(S, E, C)
例: S3 × E4 × C3 → ASIL D（最高安全要求）
```

### 3.2 AgPL（ISO 25119）

AgPL（Agricultural Performance Level）：a・b・c・d・e の5段階（eが最高安全要求）。

**ISO 25119はAgPLとSILの両方を選択できる**。ISO 26262から転身してきたエンジニアにはSIL（1〜3）で説明した方が通じやすい場合がある。

AgPLの決定もHARAベースで、類似したパラメータ（深刻度・暴露・回避可能性）を評価する。

### 3.3 ASILとAgPLの対応関係（参考）

| ASIL | AgPL | SIL |
|---|---|---|
| ASIL A | AgPL a / b | SIL 1 |
| ASIL B | AgPL b / c | SIL 1 |
| ASIL C | AgPL c / d | SIL 2 |
| ASIL D | AgPL d / e | SIL 3 |
| QM（品質管理のみ） | AgPL a以下 | ─ |

対応はあくまで参考値。正式な比較はIEC 61508ベースで行うこと。

## 4. 主要な差異の詳細

### 4.1 量産 vs 少量生産

**ISO 26262**は大量生産（自動車メーカーの年間数十万台以上）を前提とした規格。ランダムハードウェア故障率の定量的要件（FIT値、FMEDA等）が詳細に規定されており、それを満たすHW設計が求められる。

**ISO 25119**は建設機械・農業機械の少量生産（年間数十〜数千台）を考慮した構成。定量的要件は存在するが、ISO 26262ほど厳格ではなく、**既存実績（Proven-in-Use）の活用**が認められている。

### 4.2 Proven-in-Use（使用実績による安全性証明）

ISO 25119の重要な概念。長年使用されてきた部品・ソフトウェアについて、十分な使用実績と故障データがある場合、新規の安全解析・検証を省略または軽減できる。

建設機械の既存内製コードは「Proven-in-Use」の候補になり得るが、**使用実績・故障データの記録がない場合は認められない**。現状は仕様書も設計書もない状態のため、AUTOSAR移行時に新規開発として扱う必要がある。

### 4.3 プロセス要件の比較

| 要件 | ISO 26262 | ISO 25119 |
|---|---|---|
| HARA | 必須（詳細手順規定） | 必須（手順は柔軟） |
| ソフトウェアアーキテクチャ設計 | 詳細要件あり | 要件あり（簡略化可） |
| ソフトウェアユニットテスト | MC/DC網羅率等の詳細規定 | 要件あり（柔軟） |
| ハードウェア評価（FMEDA等） | 必須（定量的目標あり） | 推奨（定量的要件は緩め） |
| 独立検証 | ASIL C/Dでは独立した安全担当者が必要 | 要件あり（軽量） |
| ツール認定（TCL） | 詳細な認定プロセス | 同様の考え方を適用 |
| 機能安全マニュアル | 必須 | 必須 |

### 4.4 ソフトウェア開発の違い

ISO 26262はソフトウェア開発に対して詳細な要件を持つ（Part 6）。

| 項目 | ISO 26262（ASIL D） | ISO 25119（AgPL e） |
|---|---|---|
| 設計言語 | Cを推奨、Misra-C準拠が実質必須 | 規定なし（良好な実践を求める） |
| コードレビュー | 独立レビュー必須 | 推奨 |
| テスト網羅率 | MC/DC（Modified Condition/Decision Coverage）が原則 | 要求レベルに応じて選択 |
| 形式手法 | ASIL Dでは適用が強く推奨 | 明示的要件なし |

## 5. 委託先との協働における実務的な注意点

### 5.1 自動車業界出身エンジニアに多い誤解

- 「ASILとAgPLは直接換算できる」と思っている場合がある → あくまで参考対応であり、HARA再実施が必要
- 「ISO 26262の方が厳しいから過剰品質になる」と思って適用しようとする場合がある → ISO 25119に則ることが重要。ISO 26262の手法を参考にすることは良いが、認証はISO 25119で取得する
- 「少量生産だからProven-in-Useで済む」と思う場合がある → 記録・データが存在しなければ不可

### 5.2 委託先への仕様書・SOW（Statement of Work）記載事項

- 適用規格：ISO 25119（〇版）を明記
- 目標AgPL/SILを機能ごとに定義（HARAの結果を添付）
- Proven-in-Useを認める部品・ソフトウェアの範囲と必要な証跡を明記
- ISO 26262の知見を持つ委託先には「ISO 25119ベースで進めるが、26262のアプローチを参考にすることは歓迎」と明示する

---

## 6. AUTOSARと機能安全規格の関係

### 6.1 Classic AUTOSAR

Classic AUTOSARはISO 26262への適合を支援するよう設計されている。BSW・RTEの各モジュールは「SafetyManual（機能安全マニュアル）」が提供されており、それに従った使用がISO 26262（及びISO 25119）対応の基盤となる。

### 6.2 注意点

- AUTOSARへの準拠＝機能安全規格への適合ではない。AUTOSAR使用はISO 25119対応の「手段の一部」に過ぎない
- HARAを実施し、安全要求を定義し、AUTOSAR設定に安全要求を落とし込み、検証する、というプロセスが必要

---

## 7. まとめ

| 比較軸 | ISO 26262 | ISO 25119 |
|---|---|---|
| 対象 | 乗用車・商用車 | 農業・建設機械 |
| 安全度分類 | ASIL A〜D | AgPL a〜e（SIL 1〜3も可） |
| 量産規模 | 大量生産前提 | 少量生産対応 |
| 既存実績活用 | 限定的 | Proven-in-Use明示 |
| プロセス厳格さ | 高（特にASIL C/D） | 中（柔軟な適用可） |
| 建設機械への直接適用 | 不可（対象外） | 必須（直接適用） |

**キーメッセージ**：ISO 26262の知見を持つ委託先は貴重なパートナーだが、建設機械の認証・開発はISO 25119が正解。差異を共有した上で協力関係を構築することが重要。
