# 建設機械におけるデジタルツイン活用ガイド

## 1. デジタルツインとは（建設機械の文脈で）

デジタルツインとは、物理的な機械・現場をソフトウェア上に再現した「仮想モデル」であり、センサーデータとの双方向同期によってリアルタイムに実機の状態を反映する。

### 3つの活用フェーズ

| フェーズ | 目的 | 主な対象 |
|----------|------|----------|
| 開発 | 設計検証・テスト自動化 | ECU・油圧システム・制御ロジック |
| 製造 | 組み立て工程の最適化・品質保証 | 生産ライン・部品トレーサビリティ |
| 運用 | 予知保全・遠隔診断・施工最適化 | フリート全体・現場地形 |

### 業界での実例

- **コマツ**: KOMTRAX／Smart Construction により稼働データをクラウドに集約し、フリート全体の位置・燃料・アラートを可視化
- **Caterpillar**: Cat Digital（旧Minestar）でドローン測量データと機械の施工データを統合し、土工量を自動計算
- **日立建機**: ConSitを通じた遠隔診断サービスで、エラーコードと稼働パターンを組み合わせた故障予測を提供

## 2. 開発フェーズでのデジタルツイン

### MBD（モデルベース開発）との関係

MBDで作成したMATLAB/Simulinkモデルがデジタルツインの「物理モデル層」を構成する。制御ロジックは同一モデルからコード生成（AUTOSAR準拠）されるため、設計変更がシミュレーションと実装に同時反映される。

### SIL/HILテスト環境との連携

- **SIL（Software-in-the-Loop）**: 仮想ECUを用いて制御ソフトウェア単体を検証。実機不要で早期にバグを検出できる
- **HIL（Hardware-in-the-Loop）**: 実ECUに仮想プラントモデルを接続。油圧シミュレーターと組み合わせることでアクチュエーター応答を実機に近い条件で検証可能

### AUTOSAR開発でのシミュレーション活用（仮想ECU）

AUTOSAR Adaptive/Classicの仮想ECU（Virtual ECU）をSIL環境上で動作させることで、実機調達前からOTA更新フローやDiagnostics（UDS）の結合テストが実施できる。外部委託先とのインターフェース定義（ARXML）もシミュレーション環境で事前検証することで手戻りを削減できる。

## 3. 運用フェーズでのデジタルツイン

### フリートデータ→クラウド→デジタルツインの流れ

```
実機センサー → 車載ゲートウェイ（4G/5G） → クラウド → デジタルツイン更新 → 可視化・アラート
```

### 予知保全・故障診断への活用

- 油温・油圧・振動データをデジタルツインに反映し、物理モデルとの乖離を異常スコアとして算出
- 過去の故障パターンとの照合により、交換推奨タイミングを事前通知
- 遠隔でのDTCログ取得と組み合わせることでディーラー派遣前に原因を絞り込み可能

### 施工現場のデジタルツイン（地形・機械の統合）

ドローン測量（点群データ）と機械の施工ログを統合することで、設計図面との差分をリアルタイムに確認できる。自律施工・マシンコントロールの精度向上にも寄与する。

## 4. 実装に必要なもの

### センサーデータの種類と収集方法

| データ種別 | 収集方法 |
|-----------|---------|
| エンジン回転数・油温・油圧 | CAN/CANopen → ゲートウェイ |
| IMU（傾斜・振動） | 車載センサー → AUTOSAR SWC |
| 位置情報 | GNSS（RTK） |
| 映像・点群 | カメラ・LiDAR → エッジ処理後に送信 |

### クラウドプラットフォーム

- **AWS IoT TwinMaker**: 3Dモデルとセンサーデータを統合し、Grafanaダッシュボードと連携。既存S3/Timestream資産を活用しやすい
- **Azure Digital Twins**: OntologyベースのモデリングとADT Explorerによる関係可視化が強み。HoloLens連携による現場AR表示も可能

### 物理モデルのソフトウェア化

油圧回路はModelicaまたはSimscapeで記述し、コンテナとしてデプロイ。クラウド上でリアルタイム実機データを境界条件として与えることで、センサーのない内部状態（例：弁の摩耗度）を推定する。

## 5. 段階的な導入ロードマップ

| 時期 | マイルストーン | 具体的な施策 |
|------|--------------|------------|
| **今すぐ（〜2年）** | 稼働データ収集基盤の整備 | CAN→クラウド連携、フリート可視化ダッシュボード構築 |
| **5年後（2031年）** | リアルタイム状態監視 | 物理モデル統合、予知保全アラート、SIL/HIL環境でのOTA検証 |
| **10年後（2036年）** | 完全なサイバーフィジカルシステム | 自律施工ループ、現場デジタルツインとの統合、AI施工計画立案 |

AUTOSAR準拠プラットフォームへの移行と並行してデータ収集アーキテクチャを設計することで、開発フェーズのシミュレーション資産を運用フェーズのデジタルツインにそのまま流用できる。これが外部委託移行とSDV化を同時に進める際の最大の効率化ポイントとなる。
