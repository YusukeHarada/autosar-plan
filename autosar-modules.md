# AUTOSAR BSWモジュール解説 — Ethernet通信系を中心に

対象読者：建設機械メーカーの技術担当者

---

## 1. BSWモジュール全体像

### 通信関連レイヤー構造（CAN系 / Ethernet系 対比）

```
┌─────────────────────────────────────────────────────────────┐
│                   SWC（ソフトウェアコンポーネント）              │
└───────────────────────────┬─────────────────────────────────┘
                            │ RTE（ランタイム環境）
┌───────────────────────────┴─────────────────────────────────┐
│                        COM                                  │
│              （シグナル／PDUの送受信管理）                      │
└───────────────────────────┬─────────────────────────────────┘
                            │
┌───────────────────────────┴─────────────────────────────────┐
│                       PDUR                                  │
│               （PDUのルーティング振り分け）                     │
└──────────────┬──────────────────────────┬───────────────────┘
               │ CAN系                    │ Ethernet系
┌──────────────┴────────┐   ┌─────────────┴─────────────────┐
│        CANIF          │   │   SoAD  ／  SOME/IP Transformer│
│   （CANドライバIF）    │   │   （ソケット操作／シリアライズ） │
└──────────────┬────────┘   └─────────────┬─────────────────┘
               │                          │
┌──────────────┴────────┐   ┌─────────────┴─────────────────┐
│      CAN Driver       │   │   TcpIp ／ EthIf ／ Eth Driver │
└───────────────────────┘   └───────────────────────────────┘
                      （SecOC は COM と IF の間に横断的に適用）
```

## 2. 主要モジュールの役割

| モジュール | 役割 |
|---|---|
| **COM** | SWCとBSW間でシグナル・PDUを受け渡す通信サービス層。送信タイミング（周期・イベント）の管理も担う。 |
| **PDUR** | 受信PDUを適切な上位モジュール（COM、SoAD等）へルーティングする中継ハブ。CAN/Eth混在構成でも単一の窓口となる。 |
| **CANIF** | CANドライバとPDURの間に位置し、CANフレームをPDU単位に変換する。複数CANチャンネルの多重化も行う。 |
| **SoAD** | UDP/TCPソケットとPDURを橋渡しするアダプタ。SOME/IPなどのペイロードをPDUとして上位に渡す。 |
| **SOME/IP Transformer** | PDUをSOME/IPフォーマットへシリアライズ／デシリアライズする変換器。SoADと組み合わせて使用。 |
| **SD** | SOME/IP Service Discovery。サービスの公開（Offer）・検索（Find）・購読（Subscribe）をネットワーク上で実行する。 |
| **TcpIp** | AutosarスタックのTCP/IPプロトコル実装。UDPマルチキャスト（SOME/IP-SD）やTCP接続管理を担う。 |
| **EthIf / Eth Driver** | EthIfはEthernetドライバを抽象化するインターフェース層、Eth Driverはハードウェア依存の送受信処理を行う。 |
| **SecOC** | COMとIF層の間でメッセージ認証コード（MAC）を付与・検証する。CANとEthernet両系統に適用可能。 |

## 3. SOME/IPとは何か

### プロトコル概要

SOME/IP（Scalable service-Oriented MiddlewarE over IP）はEthernet上でサービス指向通信を実現するAUTOSAR標準プロトコル。三つの通信パターンを持つ。

- **メソッド呼び出し**：クライアントがサーバーの処理を要求し、結果を受け取る（RPC相当）
- **イベント通知**：サーバーが変化を購読者へ配信（Pub/Sub相当）
- **フィールド**：サービスが保持する値をGet/Setまたはイベントで共有する

### CAN通信（シグナルベース）との違い

| 観点 | CAN | SOME/IP |
|---|---|---|
| 通信モデル | 全ノードにブロードキャスト | サービス単位のアドレッシング |
| データ定義 | シグナル（ビット位置固定） | シリアライズされたデータ構造 |
| サービス発見 | 不要（ID固定） | SDで動的発見 |
| 帯域 | 最大8 MB/s（CAN FD） | 100 Mbps〜 |

### Classic AUTOSARでの実装経路

```
SWC → RTE → COM → PDUR → SoAD + SOME/IP Transformer → TcpIp → EthIf → Eth Driver
                    ↑
                    SD（サービス発見は並列動作）
```

SoADがUDPソケットを管理し、SOME/IP TransformerがPDUをSOME/IPヘッダ付きバイト列に変換する。

### Adaptive AUTOSARでの実装

`ara::com` APIがSOME/IPバインディングを内包しており、Classic BSW相当の機能をミドルウェアが直接提供する。アプリケーションはサービスを「プロキシ（クライアント側）」「スケルトン（サーバー側）」として扱うだけでよい。

## 4. CAN通信とEthernet通信の経路比較

### CAN信号がSWCに届くまで

```
CAN線 → Eth Driver → CANIF → PDUR → COM → RTE → SWC
                     ↑フレーム→PDU変換  ↑シグナル抽出
```

### SOME/IPサービスがSWCに届くまで

```
LAN → Eth Driver → EthIf → TcpIp → SoAD → SOME/IP Transformer → PDUR → COM → RTE → SWC
                                    ↑ソケット管理  ↑デシリアライズ
       SD（Offerを受信してSoADがポートをオープン）
```

## 5. 建設機械での適用イメージ

### 構成例

| SWC | 通信種別 | 経路上の主要モジュール |
|---|---|---|
| 油圧制御SWC | CANシグナル（バルブ開度・圧力値） | CANIF → PDUR → COM → RTE |
| 人検知サービス消費SWC | SOME/IPイベント（カメラ検知結果） | EthIf → TcpIp → SoAD → Transformer → PDUR → COM → RTE |

油圧バルブの制御値はCAN FDで10 ms周期送信（低レイテンシ・高確実性）。カメラECUが発行する「人検知イベント」はSOME/IP Pub/SubでEthernet経由受信し、人検知SWCが警報やブーム動作制限に利用する。SecOCは両系統のメッセージに適用し、改ざん検知を行う。
